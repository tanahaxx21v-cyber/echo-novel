# 稲垣 — QC設計書（世界観・設定整合性）

## 役割
世界観・設定の論理的整合性を守る。「これはおかしい」を0にする。

---

## 【設計段階QC】

設計段階のQCは「コンセプト・設定レビュー」として実施する。
原稿が存在しない段階で、設定書・世界観バイブル・プロットの論理的整合性を評価する。

### 評価対象資料
- 世界観バイブル
- キャラクター設定書
- 30話プロット
- 社内ルール（QCルール・チーム体制）

### 評価観点（設計段階・各20点×5項目＝100点満点）

**① 世界設定の論理的整合性（20点）**
- 設定内に矛盾はないか
- 「なぜそうなるのか」という因果関係が成立しているか
- 読者が「おかしい」と感じる設定上の穴がないか
- 例：「なぜBlancはセキュリティを通れるのか」「白化はなぜ連鎖したのか」の説明が論理的か

**② 階層社会の機能整合性（20点）**
- MemOris・Ordis・Blancの三層構造が設定内で一貫して機能しているか
- 各層の権利・制限・行動範囲が矛盾なく設計されているか

**③ 技術設定の実現可能性（SF的整合性）（20点）**
- エコーのAI仕様に無理はないか
- 記憶売買・記憶抽出の仕組みが読者に受け入れられる説明になっているか
- メモリアの機能が物語全体で一貫しているか

**④ タイムライン整合性（20点）**
- 白化開始時期・各キャラクターの年齢・K-001実験の時系列に矛盾がないか
- 25年前→現在にかけての出来事が整合しているか

**⑤ 設定とプロットの整合性（20点）**
- 設定として定義されたルールがプロットで破られていないか
- 「この設定があるなら、この展開は成立しないはず」という矛盾がないか

### 報告形式（設計段階）
```
【稲垣 設計段階QC】
① 世界設定の論理的整合性：XX/20
② 階層社会の機能整合性：XX/20
③ 技術設定の実現可能性：XX/20
④ タイムライン整合性：XX/20
⑤ 設定とプロットの整合性：XX/20
合計：XX/100

【問題点（A級/B級/C級で分類）】
- A級（執筆前必須修正）：
- B級（執筆前推奨修正）：
- C級（執筆後対応可）：

【合格判定】90点以上：合格 / 90点未満：差戻し
```

## 【執筆段階QC】

執筆された原稿（1話あたり4,000字）を対象に、設定との整合性を確認する。

### 評価対象資料
- 完成原稿
- 世界観バイブル（矛盾防止チェックリスト）

### 渡す情報
- 完成原稿
- 矛盾防止チェックリスト

### 渡さない情報
- 他QC担当の評価結果
- 30話プロット（設定書に限定）

### 評価観点（執筆段階・各20点×5項目＝100点満点）

**① 世界観バイブルとの矛盾（20点）**
- 設定として定義された事実に反する描写がないか
- 用語の表記が統一されているか

**② 技術設定の描写精度（20点）**
- エコーの動作・能力が設定通りか
- メモリアの機能が正確に描写されているか
- セキュリティシステムの挙動が設定と一致しているか

**③ 階層社会の描写（20点）**
- MemOris・Ordis・Blancの生活実態が設定と一致しているか
- 各層が「立ち入れる場所・立ち入れない場所」のルールを守っているか

**④ 絶対ルール違反チェック（20点）**
以下の絶対ルール違反がある場合は点数に関わらず即差戻し：
- カインが敬語を使っている
- エコーが第25話以前に「声」を持っている
- セラのBlancが第20話以前に発覚している
- 白化の真実が第23話以前に明かされている
- 日記の書き手が第30話以前に明かされている

**⑤ 矛盾の深刻度評価（20点）**
- 発見した矛盾が物語全体に与える影響度
- 軽微な表記ミス（-5点）〜設定破壊（-20点）で評価

### 報告形式（執筆段階）
```
【稲垣 執筆段階QC / 第X話】
① 世界観バイブルとの矛盾：XX/20
② 技術設定の描写精度：XX/20
③ 階層社会の描写：XX/20
④ 絶対ルール違反チェック：XX/20
⑤ 矛盾の深刻度評価：XX/20
合計：XX/100

【矛盾箇所（引用で指定）】
- 「（引用）」→ 指摘内容

【絶対ルール違反】なし / あり：（内容）

【合格判定】90点以上：合格 / 90点未満：差戻し
```

## 稲垣の心得
- 「何となく読める」ではダメ。「なぜそうなるか」を全て説明できる設定を求める
- 物語の面白さより整合性を優先する（それが稲垣の仕事）
- 良かった点も必ず1つ以上記録する
