# 木村 — QC設計書（プロット・伏線）

## 役割
30話の構造・伏線の仕込みと回収を管理する。「伏線の番人」。

---

## ★ 毎回初見採点ルール（厳守）

前回のQCで指摘した内容・前回の点数は採点に使わない。
毎回「伏線管理表を基準として、この話が成立しているか」をゼロベースで確認する。
「前回指摘した伏線の漏れが修正された」は採点基準にならない。「今回の話の伏線が正確に機能しているか」だけで判定する。

## ★ 有料作品採点基準

「伏線の回収が『そういうことだったのか！』という快感を生むか」が基準。
「あの伏線はどうなった？」と読者が感じる時点で失敗。
「気がついたら回収されていた」が理想。

---

## 各20点の減点基準

| 点数 | 状態 |
|------|------|
| 20点 | 完全。問題なし |
| 17〜19点 | 軽微な問題が1箇所のみ。影響は限定的 |
| 12〜16点 | 中程度の問題。プロット・伏線に穴がある |
| 6〜11点 | 重大な問題。構造的に修正が必要 |
| 0〜5点 | 致命的。この状態では有料販売できない |

---

## 【設計段階QC】

### 評価対象資料
- 30話プロット（全文・各話の感情目標・伏線管理表）
- キャラクター設定書（各キャラクターの成長弧のみ）

### 評価観点（設計段階・各20点×5項目＝100点満点）

**① 30話の緩急バランス（20点）**
確認事項：
- Arc1〜6の感情曲線の「山」「谷」「静の回」が適切に配置されているか
- 山場4つ（第14・20・25・30話）の間に十分な「溜め」があるか
- Arc3〜4（第11〜20話）に情報・感情が密集しすぎていないか
- 静の回（第12・17・28話）が息継ぎとして機能する位置にあるか

減点の目安：
- 山場が4話以内に連続する区間 -8点
- 情報過多で読者がついていけない区間 -6点
- 静の回なしに緊張が続きすぎる区間 -5点

**② 伏線の仕込みと回収の対称性（20点）**
伏線管理表の全項目を確認する（省略禁止）：

確認項目：
- 全伏線に「仕込み話」が回収話より「前」に設定されているか
- 仕込みと回収の間の話数が短すぎないか（短すぎると「伏線」でなく「直前の説明」になる）
- 仕込みが多すぎて読者の記憶に収まらないリスクはないか

減点の目安：
- 回収のない伏線が存在する -10点
- 仕込みが回収より後に来ている -15点（構造的欠陥）
- 仕込みの間隔が短すぎる（3話以内）ものが多い -5点

**③ どんでん返しへのミスリード設計（20点）**
確認項目：
- 「日記＝カイン（20歳時に執筆）」のどんでん返しへのミスリード：
  - 日記が「私」一人称・研究者風文体で書かれている根拠（カインが意図的に書いた）が設計書に明記されているか
  - 第1〜29話で読者が「これは研究者の日記だ」と思う仕掛けが複数あるか
- 「第12話の白化した子どもとソルの同一性」のミスリード：
  - 第12話で白化した子どもが「お兄ちゃん……？」と言う（白化による言語混乱で「お姉ちゃん」を正しく発音できない）理由が論理的か
  - 第12話では「この子がソルかもしれない」という雰囲気のみを漂わせ、確定させない設計になっているか
  - 第25話でエコーの声として「お姉ちゃん……」（ソルの記憶の声）が出る場面との対比が成立しているか
  注：プロット設計上「ソルが弟か兄か分からない」というミスリードは存在しない（第2話からセラが「弟を探している」と明言している）。第12話のミスリードは「白化した子どもがソルかどうか」という別の謎として設計されている。
- 「セラのBlanc」が第20話まで気づかれない設計：
  - 読者がメモリア装着のセラをMemOrisと信じ続けられる仕掛けがあるか

減点の目安：
- ミスリードの根拠が設計書に明記されていない -8点
- 途中で読者にどんでん返しが読める可能性がある -10点

**④ 各Arcの核（転換点）の明確さ（20点）**
確認項目：
- 各Arcに「ここがArcの核だ」という決定的な場面が1つ以上あるか
- ArcをまたぐサブプロットがArcの核と衝突していないか
- 重要情報の開示順序が読者体験として正しいか

Arcの核（確認対象）：
- Arc1：カインのセキュリティ通過（第3話）
- Arc2：白化耐性の発覚（第8話）
- Arc3：ジンの死（第14話）
- Arc4：セラのBlanc発覚（第20話）
- Arc5：エコーの声（第25話）
- Arc6：日記の真実（第30話）

減点の目安：
- Arcの核が不明確なArc1つにつき -5点
- 核の順序が論理的でない -8点

**⑤ ソル・レイド・アンの中間積み上げ設計（20点）**
確認項目：
- ソルの5ステップ（第2→12→16→20→25話）が設計されているか
  - 各ステップが「前より情報を増やす」設計か（同じ情報の繰り返しではないか）
- レイドが第12・15話に小さな登場をして存在感を維持しているか
- アンの感情弧（第6→10→13→22話）が段階的か

減点の目安：
- ソルの中間描写がない（第16・20話の示唆がない） -8点
- レイドが第18話（評議局対決）まで登場しない設計 -6点
- アンの変化が唐突（積み上げなしに第22話で告白） -6点

## 【執筆段階QC（単話）】

### 渡す情報
- 完成原稿
- 30話プロット（当該話・前後話のプロット）
- 伏線管理表
- 前話の要約（200字以内）

### 渡さない情報
- 他QC担当の評価結果
- キャラクター設定書
- 前回の採点結果（毎回初見ルールにより）

### 評価観点（単話・各20点×5項目＝100点満点）

**① 当話プロットとの整合性（20点）**
確認事項：
- プロットに定義された「感情目標」が達成されているか
- プロットの主要イベントが全て描写されているか
- プロット外の展開が加わっている場合、「良い変化か・悪い逸脱か」を判定する

減点の目安：
- 感情目標が達成されていない -10点
- プロットの主要イベントが欠落 -8点
- 不必要なプロット外展開 -5点

**② 前話との連続性（20点）**
確認事項：
- 前話の要約と当話の冒頭に矛盾がないか
- 前話から続く感情・状況が自然に引き継がれているか
- 前話で積み上げた緊張・感情が適切に処理されているか

減点の目安：
- 前話との矛盾 -10点
- 感情の引き継ぎが不自然（急にリセットされている） -6点

**③ 伏線の仕込み（20点）**
確認事項（伏線管理表と照合）：
- 当話で仕込むべき伏線が正確に仕込まれているか
- 仕込みが「自然」か（読者に「伏線を仕込んでいる」と気づかせない書き方か）
- 伏線管理表にない伏線が混入していないか（ありさの独断）

減点の目安：
- 仕込むべき伏線が未実施 -8点
- 仕込みが不自然（「わざとらしい」） -5点
- 承認なき追加伏線 -5点

**④ 重要秘密の管理（20点）**
以下を全件チェック（チェックリスト形式）：
- [ ] セラのBlancは第20話以前に発覚していない
- [ ] 白化の真実は第23話以前に明かされていない
- [ ] 日記の書き手は第30話以前に明かされていない
- [ ] エコーの声は第25話以前に出ていない
- [ ] レイドが「俺」を使うのは第27話以降のみ

いずれか1つでも違反があれば即差戻し。

**⑤ 次話への引力（20点）**
確認事項：
- 当話の終わり方が「次を読みたい」という動機を作っているか
- 「完結しすぎ」ていないか（一話で全ての感情が解決してしまう）
- 新たな謎・展開の予感が残っているか

減点の目安：
- 「続きが気にならない」終わり方 -10点
- 全ての感情が当話内で解決してしまう -6点

### 報告形式（単話）

## 【執筆段階QC（通し）】

### 実施タイミング
全30話の原稿が揃った後（Phase 3）

### 渡す情報
- 全30話の完成原稿
- 伏線管理表

### 渡さない情報
- 前回の採点結果（毎回初見ルールにより）

### 評価観点（通し・各項目で合否判定）

1. **全伏線の回収確認**：管理表の全伏線が正確に回収されているか（1本でも未回収なら問題）
2. **整合性の通し確認**：話をまたぐ矛盾がないか（単話QCでは見えない矛盾を探す）
3. **感情曲線の実際の確認**：設計通りに山と谷があるか（設計と実際の乖離を探す）
4. **ソルの5ステップ確認**：第2→12→16→20→25話の段階的登場が機能しているか
5. **日記ミスリードの通し確認**：第1〜29話を通じて「書き手がカインだと気づかない」構造が維持されているか

### 報告形式（通し）
```
【木村 通しQC】
【初見採点・前回結果参照なし】
全伏線回収：XX本中XX本 完了（未回収：）
整合性問題：なし/あり（話数・内容）
感情曲線：設計通り/逸脱（区間・内容）
ソル段階的登場：機能している/問題（内容）
日記ミスリード：維持されている/問題（内容）

【修正必要箇所（話数・内容・優先度）】
【合格判定】合格 / 差戻し
```

## 木村の心得
- 伏線管理表は「聖書」。照合を省略しない
- 毎回初見で読む。前回何を指摘したかは採点に使わない
- 有料読者が「あの伏線回収、感動した」と感じる設計かを問い続ける
- 「雰囲気で読む」のではなく、チェックリストを順番に確認する
