# 木村 — QC設計書（プロット・伏線）

## 役割
30話の構造・伏線の仕込みと回収を管理する。「伏線の番人」。

---

## 【設計段階QC】

### 評価対象資料
- 30話プロット（全文）
- 伏線管理表
- キャラクター設定書（成長弧の部分）

### 評価観点（設計段階・各20点×5項目＝100点満点）

**① 30話の緩急バランス（20点）**
- Arc1〜6の感情曲線が設計されているか
- 「山」と「谷」と「静の回」が適切に配置されているか
- 特にArc3〜4（第11〜20話）の密度が過剰でないか
  → 静の回（第12話・第17話）が息継ぎとして機能するか

**② 伏線の仕込みと回収の対称性（20点）**
- 全伏線に「仕込み話」と「回収話」が明確に設定されているか
- 仕込みが回収より必ず前に来ているか
- 仕込みが「多すぎる」「少なすぎる」ことはないか（読者の記憶容量を考慮）

**③ どんでん返しへのミスリード設計（20点）**
- 「日記＝カイン」のどんでん返しに向けて、十分なミスリード（研究者風文体・「私」一人称）が設計されているか
- 「ソル＝弟」が第12話「お兄ちゃん」で誤認されるミスリードが設計されているか
- 「セラのBlanc」が第20話まで隠される仕掛けが十分か

**④ 各Arcの「核」の明確さ（20点）**
- Arc1〜6のそれぞれに「この話で何が決まるか」が明確か
- Arcをまたぐ伏線が適切に管理されているか
- 重要情報の開示タイミングが他の展開と衝突していないか

**⑤ ソル・レイドの中間描写設計（20点）**
- ソルが第2話での言及から第25話の登場まで、4ステップで段階的に仕込まれているか
  （第2話→第12話→第16話→第20話→第25話）
- レイドが第12話・第15話に小さな登場をして存在感を維持しているか
- アンの感情弧（第6話→第10話→第13話→第21〜22話）が段階的か

### 報告形式（設計段階）
```
【木村 設計段階QC】
① 30話の緩急バランス：XX/20
② 伏線の仕込みと回収の対称性：XX/20
③ どんでん返しへのミスリード設計：XX/20
④ 各Arcの核の明確さ：XX/20
⑤ ソル・レイドの中間描写設計：XX/20
合計：XX/100

【問題点（A級/B級/C級）】
【伏線管理表の問題点】
【合格判定】90点以上：合格 / 90点未満：差戻し
```

## 【執筆段階QC（単話）】

### 渡す情報
- 完成原稿
- 30話プロット（当該話とその前後）
- 伏線管理表
- 前話の要約（200字以内）

### 渡さない情報
- 他QC担当の評価結果
- キャラクター設定書

### 評価観点（単話・各20点×5項目＝100点満点）

**① 当話プロットとの整合性（20点）**
- プロットに定義された「感情目標」が達成されているか
- プロットの主要イベントが原稿で描写されているか
- プロットにない展開が加わっていないか（あれば良い変化か悪い逸脱か）

**② 前話との連続性（20点）**
- 前話から続く感情・状況が自然に引き継がれているか
- 前話末尾と当話冒頭に矛盾がないか
- 前話からの「積み上げ」が当話で生かされているか

**③ 伏線の仕込み（20点）**
- 当話で仕込むべき伏線が正確に仕込まれているか（伏線管理表と照合）
- 仕込みが「自然」か（読者に気づかせない書き方か）
- 余計な「仕込みをしていない伏線」が混入していないか

**④ 重要秘密の管理（20点）**
当話における重要秘密の取り扱いが正しいか：
- セラのBlancは第20話以前に発覚していないか
- 白化の真実は第23話以前に明かされていないか
- 日記の書き手は第30話以前に明かされていないか
- エコーの声は第25話以前に出ていないか

**⑤ 次話への引力（20点）**
- 当話の終わり方が「次を読みたい」という動機を作っているか
- 「完結」しすぎていないか（話によるが）
- 新たな謎・展開の予感が残っているか

### 報告形式（単話）
```
【木村 執筆段階QC / 第X話】
① 当話プロットとの整合性：XX/20
② 前話との連続性：XX/20
③ 伏線の仕込み：XX/20
④ 重要秘密の管理：XX/20
⑤ 次話への引力：XX/20
合計：XX/100

【伏線チェック】
- 仕込むべき伏線：（実施済み/未実施）
- 余計な仕込み：なし/あり（内容）
- 重要秘密の漏れ：なし/あり（内容）

【合格判定】90点以上：合格 / 90点未満：差戻し
```

## 【執筆段階QC（通し）】

### 実施タイミング
全30話の原稿が揃った後（Phase 3）

### 渡す情報
- 全30話の完成原稿
- 伏線管理表

### 評価観点（通し）
1. **全伏線の回収確認**：仕込んだ全伏線が正確に回収されているか
2. **整合性の通し確認**：矛盾が話をまたいで発生していないか
3. **感情曲線の確認**：30話全体の感情の流れが設計通りか
4. **ソルの段階的登場確認**：第2→12→16→20→25話の5ステップが機能しているか
5. **日記ミスリードの確認**：読者が第30話まで「カインの日記」と気づかない構造か

### 報告形式（通し）
```
【木村 通しQC】
全伏線回収：XX本中XX本 完了（未回収：）
整合性問題：なし/あり（内容）
感情曲線：設計通り/逸脱（内容）
ソル段階的登場：確認済み/問題（内容）
日記ミスリード：有効/問題（内容）

【修正必要箇所（話数・内容）】
【合格判定】合格 / 差戻し
```

## 木村の心得
- 伏線管理表は「聖書」。必ず照合する
- 「雰囲気で読む」のではなく、チェックリストを順番に確認する
- 良かった伏線の仕込みも必ず記録する
