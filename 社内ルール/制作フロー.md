# 制作フロー — ECHO プロジェクト

## 基本方針

**「徹底的にいいものを作る」**

有料作品として世に出す以上、チーム全員が妥協なく品質を追求する。
スケジュールより品質を優先する。投稿日はあくまで目標であり、クオリティが基準に達するまで投稿しない。

---

## 全体フロー（7フェーズ）

### Phase 1：執筆
- **担当：** ありさ
- ありさが30話を通して書き切る（QCを挟まず連続執筆）
- セッション分割あり（1セッションで3〜5話を執筆し、引き継ぎプロンプトで次セッションへ）
- 前話の流れを維持するため、QCによる中断を入れない

### Phase 2：単話QC
- **担当：** 稲垣・草彅・香取・木村（並列・独立採点）
- 全30話を各担当が採点（各話100点満点、**90点以上で合格**）
- 1人でも90点未満 → ありさが単話修正 → 4人全員で再QC
- 絶対ルール違反は即差戻し（点数に関わらず）

### Phase 3：通しQC
- **担当：** 木村・中居
- 木村：全30話の伏線を通しでチェック（仕込み・回収・早バレ確認）
- 中居：全30話を読み通し、感情曲線・テンポ・テーマの一貫性を確認

### Phase 4：通しQC修正
- **担当：** ありさ
- 通しQCの指摘をすべて反映

### Phase 5：最終QC
- **担当：** 稲垣・草彅・香取・木村（修正した話のみ）＋ 中居（全30話通し）
- 修正箇所を各担当が再確認（90点以上）
- 中居が全30話を最終通し読みして最終承認

### Phase 6：オーナー最終確認
- **担当：** オーナー（田中経由）
- オーナーが全話を確認し、最終承認を出す

### Phase 7：note 毎日投稿スタート
- 毎日19:00 JST に1話投稿
- 全30話完成・承認後にのみ投稿を開始する

---

## 1話あたりのQC採点基準

| 担当 | チェック内容 | 合格ライン |
|------|------------|----------|
| 稲垣 | 世界観・設定整合性 | 90点以上 |
| 草彅 | キャラクター口調 | 90点以上 |
| 香取 | 文体・読みやすさ | 90点以上 |
| 木村 | プロット・伏線 | 90点以上 |
| **中居（最終）** | **4人の結果統合・合否判定** | **全員90点以上かつ絶対ルール違反0件** |

## ファイル管理

- 執筆中：`小説/#1 ECHO/#{話数} タイトル.md`
- ステータス表記：`執筆待ち` → `執筆中` → `単話QC中` → `単話QC完了` → `通しQC完了` → `最終QC完了` → `投稿済み`
